# 📝 Reporte de Corrección NGINX

**Fecha:** 5/3/2026, 10:42:36  
**Contenedor:** nginx  
**Servicio:** nginx  
**Archivo:** nginx.conf

## 📋 Cambios Realizados

Se realizaron 4 cambio(s):

1. REMOVED duplicate 'proxy_set_header Connection $connection_upgrade' and 'proxy_set_header Upgrade $http_upgrade' from the main app location block (/new/ConfigurationApp/). The duplicate Connection header was overriding the keep-alive Connection "" directive needed for NTLM/Kerberos authentication. WebSocket upgrade is already handled by the dedicated /new/App/_blazor location block.
2. FIXED proxy_redirect: changed 'proxy_redirect https://server1.test.com/new/App/ /new/App/' to 'proxy_redirect https://server1.test.com/new/App/ /new/ConfigurationApp/' so backend redirects are correctly rewritten to the external-facing path.
3. REMOVED redundant 'proxy_redirect /new/App/ /new/App/' which was a no-op (replacing a path with itself).
4. FIXED root redirect: changed 'return 301 https://$host/new/App/' to 'return 301 https://$host/new/ConfigurationApp/' so the root URL redirects to the correct external-facing application path.
