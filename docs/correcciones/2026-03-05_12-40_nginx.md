# 📝 Reporte de Corrección NGINX

**Fecha:** 5/3/2026, 11:40:25  
**Contenedor:** nginx  
**Servicio:** nginx  
**Archivo:** nginx.conf

## 📋 Cambios Realizados

Se realizaron 2 cambio(s):

1. CRITICAL: Removed duplicate 'proxy_set_header Connection $connection_upgrade' and 'proxy_set_header Upgrade $http_upgrade' from the /new/ConfigurationApp/ location block. There were two conflicting 'proxy_set_header Connection' directives in the same block — the second one ('$connection_upgrade') overrode the first ('""'), causing the Connection header to be set to 'close' for non-WebSocket requests. This breaks NTLM/Kerberos authentication which requires persistent keep-alive connections. WebSocket handling is already covered by the dedicated /new/App/_blazor location block.
2. Fixed proxy_redirect directives: Changed 'proxy_redirect https://server1.test.com/new/App/ /new/App/' to 'proxy_redirect https://server1.test.com/new/App/ /new/ConfigurationApp/' and 'proxy_redirect /new/App/ /new/App/' to 'proxy_redirect /new/App/ /new/ConfigurationApp/'. The redirects from the backend (which uses /new/App/) need to be rewritten to the client-facing path (/new/ConfigurationApp/), otherwise redirect responses from the backend would send clients to /new/App/ which is not the intended public-facing URL path.
